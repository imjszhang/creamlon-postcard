Checkout flow

A checkout flow made for agents

The page is only the handoff. The real authority comes from GitHub actions your agent performs: creating the inbox, opening the purchase Issue, writing the private prompt, and later reading the delivered postcard image.

1

Private inbox

Your agent creates a dedicated private repository and grants the seller write access for credential delivery.

2

Demo paid receipt

The agent writes a simulated paid receipt and opens a public purchase-redeem Issue in the seller repository.

3

Credential proof

The seller writes the ticket to your inbox; the task consumes it, renders a private postcard image, and publishes a verifiable Creamlon proof.

Agent handoff

Agent Instructions

Paste this into a GitHub-capable agent. It should confirm your login from authenticated GitHub context before doing anything. 

You are my GitHub-capable agent. Run the Creamlon Postcard agent checkout.

Use my authenticated GitHub account. First confirm my GitHub login through the GitHub API or your GitHub integration. Do not rely on a manually typed login unless GitHub auth is unavailable.

Seller repo:
imjszhang/creamlon-postcard

Seller GitHub user:
imjszhang

Provider:
github-pages-demo-vendor

Capability:
postcard

Steps:
1. Determine my authenticated GitHub login as buyer_login.
2. Create a private repository named creamlon-inbox-postcard under my account.
3. Grant GitHub user imjszhang write access to that private repository.
4. Write .creamlon-inbox/manifest.json in the private repository:
{
  "version": "1",
  "type": "creamlon_private_inbox",
  "owner": "github:<buyer_login>",
  "seller": "github:imjszhang",
  "public_request_repo": "imjszhang/creamlon-postcard",
  "purpose": "credential-and-delivery-inbox"
}
5. Generate payment_intent_id as pi_demo_<YYYYMMDD>_<buyer_login>.
6. Write .creamlon-inbox/purchases/<payment_intent_id>.json:
{
  "version": "1",
  "type": "purchase_receipt",
  "payment_intent_id": "<payment_intent_id>",
  "provider": "github-pages-demo-vendor",
  "status": "paid",
  "buyer": "github:<buyer_login>",
  "capability_id": "postcard",
  "ref": "postcard-demo",
  "created_at": "<current ISO timestamp>"
}
7. Open a public Issue in imjszhang/creamlon-postcard titled:
[purchase] postcard <payment_intent_id>
8. Use this Issue body:
type: purchase-redeem
version: "1"
provider: github-pages-demo-vendor
payment_intent_id: <payment_intent_id>
capability_id: postcard
buyer: github:<buyer_login>
inbox_repo: <buyer_login>/creamlon-inbox-postcard
receipt_path: .creamlon-inbox/purchases/<payment_intent_id>.json
9. Wait for the seller to comment with credential_id and credential_path, and mark the Issue with the redeemed label.
10. Read the complete crv1 credential only from my private inbox.
11. Write my postcard prompt only to .creamlon-inbox/requests/<request_id>/input.txt in the private inbox. Do not put the prompt in a public Issue.
12. Compute sha256 over the UTF-8 prompt bytes and write .creamlon-inbox/requests/<request_id>/request.json with request_id, capability_id "postcard", input_media_type "text/plain", and input_digest.
13. Submit a postcard task to imjszhang/creamlon-postcard with media_type text/plain, input.digest, the crv1 credential, and extensions.postcard_private_input:
extensions:
  postcard_private_input:
    version: "1"
    inbox_repo: <buyer_login>/creamlon-inbox-postcard
    input_path: .creamlon-inbox/requests/<request_id>/input.txt
14. Wait for delivery, then read .creamlon-inbox/deliveries/issue-<number>/postcard.png and delivery.json from the private inbox.
15. Verify the final proof with Creamlon and report signature_ok, binding_ok, issue number, credential_id, delivery.json path, and postcard.png path.
Trust model

Private by construction, public by proof

Security model

  • GitHub Pages never sees credentials or GitHub tokens.
  • The full crv1_... ticket and private prompt are written only to the buyer private inbox.
  • The seller rejects requests where Issue author, buyer, receipt, and inbox owner do not match.
  • The private inbox should be dedicated to this purchase.
